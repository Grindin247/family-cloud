## 2026-03-21

- Investigated Caleb's hourly FamilyCloud inbox cron failure: recent runs in `~/.openclaw/cron/runs/0a4d3a4f-1e13-4c2d-8fb3-9a5d48c67c32.jsonl` were failing at startup with `ModuleNotFoundError: No module named 'agents'`.
- Root cause: `scripts/nextcloud_para_agent.py` imports `agents.common.file_inbox`, but when Python runs the script directly, `sys.path[0]` resolves to `/scripts` rather than the repo root, so the top-level `agents/` package is not importable.
- Fixed by bootstrapping the repo root onto `sys.path` at the top of `scripts/nextcloud_para_agent.py` before importing `agents.common.*`.
- Added a regression test in `tests/test_nextcloud_para_agent.py` that runs `python /abs/path/scripts/nextcloud_para_agent.py --help` from the `scripts/` directory to verify standalone startup works.
- Hardened `~/.openclaw/cron/jobs.json` so the hourly inbox-processing job now uses absolute paths for both the FamilyCloud virtualenv activation and the `nextcloud_para_agent.py` script.
- Verified with `pytest -q tests/test_nextcloud_para_agent.py` and with a live `process-ready --summary-json` run from outside the repo workspace; the live run completed and processed 2 inbox items.
